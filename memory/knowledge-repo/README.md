## What Goes Here

### Values & Beliefs
**Why**: Content needs your values integrated, not just facts
**Format**: Core principles, medical philosophies, business beliefs
**Example**: "Root cause treatment > symptom management"

### Positioning
**Why**: Every doctor is unique - capture what makes them different
**Format**: Specialty, unique approach, differentiators
**Example**: "First practice to focus exclusively on autonomic dysfunction in long COVID"

### Frameworks
**Why**: Mental models make complex concepts accessible
**Format**: Explanatory frameworks, analogies, teaching approaches
**Example**: "Autonomic system = gas pedal (sympathetic) + brake (parasympathetic)"

### Learning
**Why**: System improves over time by capturing what works
**Format**: Insights from `/learn` command after successful projects
**Grows**: Automatically as you complete work and run `/learn`

### Ideas Inbox
**Why**: Quick capture so thoughts don't get lost
**Format**: Raw inputs, incomplete thoughts, concepts to explore
**Review**: Weekly processing into structured knowledge

## Usage Pattern

### Capturing Valuable Thoughts
**When you have an insight:**
```bash
/ask "Capture this thought in my knowledge repo: [your insight]"
```

System will:
- Categorize (value/belief, framework, idea)
- Store in appropriate location
- Make searchable for future use

### Using During Content Creation
**System automatically:**
- Loads doctor's values/beliefs
- Applies relevant frameworks
- Integrates positioning
- Makes content feel authentic

**You can also:**
```bash
/recall "autonomic dysfunction frameworks"
```

### Weekly Processing
1. Review `ideas/inbox/` for unprocessed thoughts
2. Categorize and file properly
3. Extract patterns from recent work
4. Update frameworks with new insights
